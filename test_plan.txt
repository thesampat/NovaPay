NovaPay Edge-Case Testing Plan
==============================

To-Do List for k6 / Integration Testing:

[ ] 1. Payroll Processing
    - Test sending a single `payroll/process` request with a huge `paylist` array.
    - Verify that BullMQ successfully streams and handles each recipient async.
    - Verify final balances for the Employer (Sender) and all Employees (Receivers).

[ ] 2. Sender-Receiver Multiple Concurrent Requests
    - Send 50 simultaneous transfer requests from Account A to Account B.
    - Ensure MongoDB locking mechanisms prevent race conditions (overdrafts or lost increments).

[ ] 3. Idempotency Handling
    - Send the EXACT same `transactionId` twice in a row.
    - Ensure the transaction is only processed once (deduplicated), preventing the sender from being double-charged.

[ ] 4. Ledger Failure (Ledger Miss)
    - Simulate a failed Ledger insertion (e.g. invalid status or network failure) during the 6-leg dual entry processing phase.
    - Validate that the Transaction Service correctly catches the error, updates whatever ledger entries it *did* write to 'FAILED', and doesn't actually remove money from the user's wallet.

[ ] 5. FX Failure 
    - Simulate the FX service crashing, returning an invalid rate, or clearing the rate unexpectedly during a live transaction.
    - Ensure the Gateway returns an appropriate 4xx/5xx code without touching balances or writing corrupt Ledger entries.
